1、订单编号：从订单取

2、License ： 从订单取

3、设备企业 ： License 所属企业

4、分成上级：分成上级企业

5、下单应用 ：从订单取

6、套餐名称：从订单取

7、订单状态：已支付（用户付款）、已退款（订单发生退款）

8、退款/付款时间：订单支付时间/退款时间 对应 下单和退款业务

9、支付币种：订单取

10、订单支付金额：付款 = 正数，退款 = 负数

11、支付费率：企业自己维护（在“收款账户”中取，若不存在即为0）

12、支付手续费：支付金额 * 支付费率

13、套餐成本：即套餐底价

       13.1、平台公版App：套餐底价 = “-”

       13.2、企业自有App && 平台收款 && 直接分成：套餐底价 = “-”

       13.3、企业自有App && 平台收款 && 先扣底价再分成：套餐底价 = 平台定义底价（注意平台是否对该企业自定义过底价）

       13.4、企业自有App && 企业收款：套餐底价 = 平台定义底价（注意平台是否对该企业自定义过底价）

14、收入（可分）金额 = 订单支付金额 - 支付手续费 - 套餐底价

15、分出金额 = 按照对外分成比例分出去的多个企业汇总,点击查看分出的企业，点击跳转到《合作商分成、分出明细》携带条件 订单编号 = XXXX

16、结余金额：可分成金额 - 已分出金额




注意多套餐拆分成多条，历史算法未变动。分出金额 = 设备套餐底价/组合套餐底价 * 可分成金额(扣除手续费、扣除套餐底价) * 分成比例
